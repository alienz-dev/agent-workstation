How It Helps Developers

Agent Workstation gives developers a complete AI-native development environment from a single CLI. Instead of learning 4+ tools and managing multiple databases, you get one unified interface that handles spawning agents, enforcing quality pipelines, tracking issues, learning from failures, and coordinating multi-agent work — all from your terminal.

Developer Workflow: From Task to Done

1. aw init

Scaffold project with SDD methodology, constitution, roles, and quality gates. One command sets up the full AI-native environment.

2. aw spawn planner "design feature X"

Planner agent writes spec with EARS acceptance criteria. Information barrier: coders never see the spec — tests ARE the spec.

3. Pipeline: test → sprint → review

Test-manager generates failing tests (RED gate). Sprint-manager dispatches coders in parallel waves. Reviewer does adversarial review.

4. Quality gates enforce standards

Machine-checkable gates at each transition. No agent claim without evidence. Pipeline blocks until verification command passes.

5. aw heuristic — learn from failures

Post-session reflection extracts lessons. Next session's agents get relevant heuristics in their briefings. System gets smarter over time.

Agent-Agnostic Protocol

Any coding agent that can read a markdown briefing and write a markdown result can participate. Native runtime (kong loop) for maximum integration, or adapters for external agents. Aider gets --message with the task. Claude-code gets --print. The pipeline doesn't care what's inside the pane — it only cares about the result file.

For Developers: What Changes

Install agent-workstation globally. Run aw init in your project. You get: a coding agent that follows SDD methodology, enforces quality gates, learns from its mistakes, recovers from crashes, works with multiple LLM providers, and coordinates multi-agent plans — all from one aw command. No more juggling 4 CLIs. No more lost state. No more starting every session from scratch.

Implementation Phases

Phase 1: Core Foundation

Package structure, agent protocol types, session client, unified DB schema, plugin interface.

Phase 2: Runtime

Kong agent loop → @agent-workstation/runtime. Observation masking, multi-provider, checkpoints.

Phase 3: Orchestration

Plan system, heuristics, DAG dispatch, error classification, circuit breaker.

Phase 4: Workflow & Quality

Pipeline FSM, configurable gates, SDD/TRIO enforcement, review tiers.

Phase 5: Knowledge & Plugins

Issues, knowledge graph, browser, devops, remote sync.

Phase 6: Agent Adapters

Native (kong), kiro, aider, claude-code, generic file-based.
