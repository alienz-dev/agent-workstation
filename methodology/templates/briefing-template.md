# Briefing Template

Standard format for all agent briefings. Generated by `aw spawn`.

```markdown
## Task
[1-3 sentences: clear goal. What "done" looks like.]

## Context
[Inlined essentials — code snippets, error messages, decisions. ≤5KB.]

## Read Directives (when content >5KB)
- `path` — structural context (what to look for, WHY it matters)

## Constraints
- Owned files: [exclusive write access list]
- Do NOT: [explicit anti-patterns]
- Scope: [what's in/out]

## Verification
Command: `[exact command to run]`
Expected: [pattern or exit code]

## Relevant Lessons
- [heuristic 1: trigger → action]
- [heuristic 2: trigger → action]
```

## Rules

- **Inline ≤5KB** — code snippets, errors, decisions
- **Read Directive >5KB** — path + WHY (not bare paths)
- **Always inline:** error messages, constraints, architectural decisions
- **No placeholders** — no TBD, no TODO, no "add appropriate"
- **Verification is prescriptive** — exact command + expected output
- **Heuristics max 3** — most relevant from FTS5 match
- **Coder briefings:** tests-as-contract. NO spec text. NO spec paths.
